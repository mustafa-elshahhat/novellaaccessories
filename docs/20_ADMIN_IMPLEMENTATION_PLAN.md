# Admin Implementation Plan — Current Architecture

The admin information architecture is task-oriented and has exactly 11 sidebar destinations:

```text
Dashboard
Products
Categories
Orders
Customers
Discounts
Storefront Content
Shipping
WhatsApp
Expenses
Reports
```

Canonical ownership:

- Products own product content, pricing, variants, stock, images, active/featured state, and optional product search appearance fields.
- Categories own category content, image, ordering, active state, and optional category search appearance fields.
- Discounts owns coupons and the two-delivered-orders reward settings.
- Storefront Content owns homepage heroes and the seven fixed static pages. Page keys/routes are fixed; slugs, canonical URLs, hreflang, and indexing are generated by the application.
- Shipping owns governorates, customer-paid fees, actual shipping cost, active/sort state, margin behavior, and the free-shipping rule.
- WhatsApp owns connection/session operations through the API proxy, editable business templates, reminder automations, logs, retries, and test sends.
- Dashboard owns read-only operational status for payment readiness, WhatsApp status/failures, and SEO coverage counts.
- Reports owns all reporting and analytics, including the analytics funnel.

Environment-managed configuration:

- Domain/site URL and default brand SEO fallback are code/environment managed for the single-brand Novella store.
- WhatsApp sidecar base URL and internal key are environment managed through `WhatsApp__BaseUrl` and `WhatsApp__InternalApiKey`.
- Payment provider configuration remains environment managed and read-only in admin status cards.

Removed standalone admin pages:

```text
/analytics
/seo
/settings
/payments
/heroes
/pages
/whatsapp/settings
/whatsapp/logs
/coupons/two-order-settings
```

Database migration:

- `AdminInformationArchitectureRedesign` creates `ShippingSettings`, migrates free-shipping values from `SiteSettings`, drops `SiteSettings`, and drops obsolete WhatsApp fixed-template/base-URL columns.
